What is event loop? How does it work?

JavaScript is JIT Compiled, Multi-Paradigm, prototyped-based Programming Language. JavaScript is "Single Threaded" and synchronous Programming Language by nature. Single Threaded means only one statement is executed at a time. JavaScript only has one call stack. JavaScript runs code line by line. Must finish executing a piece of code before moving onto the next.
But also JavaScript Smoothly handled asynchronous behaviour and workflow. JavaScript has a mechanism that handled asynchronous workflow and callback function and stack and queue is called event loop. Its a Js model and mechanism to handle executing the code, processing events and handle and executing the callback functions(stack and queue).

When there are tasks in the call stack, they are going to be executed and therefore the call stack is empty. when the call stack is empty, the event loop is going to check if there is any task in the queue(event queue, scheduled queue, microtask queue). If there is a task in the event queue, the event loop pass the task. if there is not a task in the event queue, the event loop is going to see on the scheduled queue, if there's a task scheduled, the event loop will pass that task. then, it's going to see if there's a task in the microtask queue, if there's a task, the event loop will pass the task.
